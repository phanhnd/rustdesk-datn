### 7c. Block User Access — Kiểm tra quyền trước khi kết nối

**Mục tiêu**: Ngăn user kết nối đến máy họ không được phép.

#### Luồng hoạt động
```
[User nhập ID → nhấn Connect]
    ↓
[index.tis: createNewConnect()]
    ↓
[handler.check_access_blocking(rustdesk_id)]  ← gọi Rust
    ↓
[src/ui.rs: UI::check_access_blocking()]
    ↓
POST http://127.0.0.1:3000/api/check-access
{ rustdesk_id: "...", Authorization: "Bearer <token>" }
    ↓
[server.js: /api/check-access handler]
    ↓
Logic:
  - Máy không trong hệ thống → allowed: true (failopen)
  - Không có token → allowed: false, reason: "login_required"
  - Có token → decode JWT, kiểm tra roles → allowed: true/false
    ↓
[Nếu không cho phép → msgbox lỗi, KHÔNG kết nối]
[Nếu cho phép → handler.new_remote(id, type, force_relay)]
```

#### Code Rust (`src/ui.rs:497–528`)
```rust
fn check_access_blocking(&mut self, rustdesk_id: String) -> String {
    use hbb_common::config::LocalConfig;
    let token = LocalConfig::get_option("access_token");
    let client = reqwest::blocking::Client::builder()
        .timeout(std::time::Duration::from_millis(800))
        .build()
        .unwrap_or_else(|_| return "".to_owned()); // failopen
    let mut builder = client
        .post("http://127.0.0.1:3000/api/check-access")
        .json(&serde_json::json!({ "rustdesk_id": rustdesk_id }));
    if !token.is_empty() {
        builder = builder.header("Authorization", format!("Bearer {}", token));
    }
    match builder.send() {
        Ok(resp) => match resp.json::<serde_json::Value>() {
            Ok(v) => {
                if v["allowed"].as_bool().unwrap_or(true) { return "".to_owned(); }
                match v["reason"].as_str().unwrap_or("no_permission") {
                    "login_required" => "Bạn cần đăng nhập để kết nối máy này".to_owned(),
                    _ => "Bạn không có quyền truy cập máy này".to_owned(),
                }
            }
            Err(_) => "".to_owned(), // failopen
        },
        Err(_) => "".to_owned(), // server offline → failopen
    }
}
```

#### Code TIS (`src/ui/index.tis:100–118`)
```js
function createNewConnect(id, type) {
    // ...
    var err = handler.check_access_blocking(id);
    if (err) {
        msgbox("custom-error", "Không có quyền truy cập", err);
        return;
    }
    handler.set_remote_id(id);
    handler.new_remote(id, type, force_relay);
}
```

#### Dispatch registration (`src/ui.rs:779`)
```rust
fn check_access_blocking(String);  // đã đăng ký trong dispatch_script_call!
```

**Failopen policy**: Server offline, timeout, parse lỗi → **cho phép kết nối** (không block oan)

### 7d. Keycloak Login + Address Book tích hợp

**Mục tiêu**: User login qua Keycloak OAuth2, lấy address book từ gateway theo role.

#### Login flow (`src/ui/index.tis:1527–1574`)
```
[Nhấn Login trong menu]
    ↓
login() → msgbox username/password
    ↓
POST {api_server}/api/login
{ username, password, id, uuid, type: 'account', deviceInfo }
    ↓
Nếu type == 'email_check' → on_2fa_check() (2FA flow)
    ↓
Nếu thành công:
  handler.set_local_option("access_token", data.access_token)
  set_local_user_info(data.user)
  getAb()  → lấy address book
```

#### Address book fetch (`ab.tis`)
```
POST /api/address-books
Authorization: Bearer <access_token>
    ↓
server.js decode JWT → lấy roles → getMachinesForRoles()
→ trả về { machines: [...] }
    ↓
ab.tis populate danh sách machines trong tab Address Book
```

#### Token storage
- `access_token` lưu trong `LocalConfig` qua `handler.set_local_option("access_token", ...)`
- `user_info` lưu JSON `{name, display_name, avatar, status}`
- Kiểm tra khi startup: `refreshCurrentUser()` → nếu token invalid → `reset_token()`

## 13. Kiến thức bổ sung

### Sciter TIS call pattern
```js
// TIS gọi Rust
handler.methodName(arg1, arg2)

// Rust trả về String rỗng "" = thành công / no-op
// Rust trả về String có nội dung = thông báo lỗi
```

### IPC pattern
```rust
// UI → background service
ipc::get_id()
ipc::get_config("key")
```

### Failopen security policy
- `check_access_blocking` trả `""` (empty = allowed) khi:
  - Server timeout (800ms)
  - Server offline
  - Parse JSON lỗi
- Lý do: tránh block oan khi gateway không chạy

### Token trong KC JWT
```js
// Decode JWT payload để lấy roles
payload.realm_access.roles[]           // realm roles
payload.resource_access[CLIENT_ID].roles[]  // client roles
```
